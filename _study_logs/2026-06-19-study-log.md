## 学んだこと
### Git のユーザー情報を確認する

現在設定されている Git のユーザー名やメールアドレスは、以下のコマンドで確認できる。

```bash
git config user.name
git config user.email
```

すべての設定を一覧で確認したい場合は、以下のコマンドを使う。

```bash
git config --list
```

### グローバル設定

普段使う名前やメールアドレスは、グローバル設定として登録しておく。

```bash
git config --global user.name "Your Name"
git config --global user.email "your@example.com"
```

`--global` を付けると、その PC 上の Git 全体に対する設定になる。

グローバル設定は、基本的には以下のファイルに保存される。

```bash
~/.gitconfig
```

Windows の場合も、Git Bash や WSL では `~/.gitconfig` として扱われる。

### リポジトリ単位の設定

プロジェクトごとに別の名前やメールアドレスを使いたい場合は、対象のリポジトリ内で `--global` を付けずに設定する。

```bash
git config user.name "Project Name"
git config user.email "project@example.com"
```

この場合、設定はそのリポジトリ内の以下のファイルに保存される。

```bash
.git/config
```

リポジトリ単位の設定は、グローバル設定よりも優先される。

### 設定の優先順位

Git config には、大きく分けて以下のような設定範囲がある。

```text
system  : PC 全体の設定
global  : ユーザー単位の設定
local   : リポジトリ単位の設定
```

基本的には、より限定的な設定が優先される。

```text
local > global > system
```

そのため、普段使う設定は `--global` に登録しておき、プロジェクトごとに変更したい場合だけローカル設定を追加するのがよさそう。

### .gitconfig の書き方

`.gitconfig` は以下のような形式で書かれている。

```ini
[user]
    name = Your Name
    email = your@example.com
```

コマンドで設定しても、最終的にはこのような形で設定ファイルに反映される。

直接ファイルを編集しても設定できるが、基本的には `git config` コマンドで設定した方が安全だと感じた。
