## 今日やったこと
直前のコミットを修正するために、`git commit --amend` の使い方を確認した。

コミットメッセージだけを修正したい場合は、次のコマンドを実行する。

```bash
git commit --amend
```

エディタが開くので、直前のコミットメッセージを修正して保存する。

エディタを開かずに、コマンド上でコミットメッセージを直接指定することもできる。

```bash
git commit --amend -m "修正後のコミットメッセージ"
```

ファイルの修正内容も直前のコミットに含めたい場合は、先に変更したファイルを `git add` してから `--amend` を実行する。

```bash
git add 修正したファイル
git commit --amend
```

コミットメッセージも同時に変更したい場合は、次のように `-m` を付ける。

```bash
git add 修正したファイル
git commit --amend -m "修正後のコミットメッセージ"
```

## 学んだこと
`git commit --amend` は、直前のコミットを作り直すためのコマンド。

主に次のような場面で使える。

- 直前のコミットメッセージを修正したいとき
- 直前のコミットにファイルを追加し忘れたとき
- 直前のコミット内容を少しだけ修正したいとき

`--amend` は「修正する」「改める」という意味で、Git では直前のコミットを新しい内容で置き換える操作になる。

`-m` を付けると、エディタを開かずに新しいコミットメッセージを指定できる。

```bash
git commit --amend -m "hogehoge"
```

そのため、コミットメッセージだけを手早く変更したい場合は `-m` を使うと便利。

ただし、まだ push していないコミットで使うのが基本。

すでに push 済みのコミットに対して `--amend` を使うと、リモート上の履歴とローカルの履歴がずれるため注意が必要。
